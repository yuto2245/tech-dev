# Sandbox Screenshot Checklist

このドキュメントは、README.md の手順を完了させたあとに、noVNC 上で Linux デスクトップを操作してスクリーンショットを撮影するための手順をまとめたものです。ユーザーに成果物として提示する場合は、以下の手順に沿って 5 枚の画像を取得してください。

## 前提条件

- README.md に従ってバックエンドとフロントエンドを起動していること。
- ブラウザから `http://localhost:5173` にアクセスできること。
- サンドボックスが `SANDBOX_PASSWORD` に設定したパスワードで noVNC 接続を受け付けていること。
- ローカルマシンにスクリーンショット用ツール（例: macOS の `Cmd + Shift + 4`, Windows の `Win + Shift + S`, Linux の `gnome-screenshot` や `flameshot`）がインストールされていること。

## 取得するスクリーンショット

1. **noVNC ログイン画面**
   - フロントエンドの「仮想環境を表示」ボタンを押し、モーダル内に noVNC 画面が表示された状態を撮影します。
   - パスワード入力ダイアログが開いた状態で撮影すると、ログイン前の状態であることが示せます。

2. **Linux デスクトップ初期画面**
   - パスワードを入力し、XFCE デスクトップが表示されたらモーダル全体を撮影します。
   - 画面左上にあるアプリケーションメニューやタスクバーが含まれるようにしてください。

3. **ターミナルで `ls` を実行**
   - デスクトップ上でターミナル（`xfce4-terminal` など）を起動します。
   - `ls` コマンドを実行し、ホームディレクトリの内容が表示された状態を撮影します。

4. **vim でテキストファイルを編集**
   - ターミナルで `vim sample.txt` を開き、`i` キーで挿入モードに入ります。
   - 任意の文章（例: `Hello from the sandbox.`）を入力した状態で、ターミナルウィンドウが見えるように撮影します。

5. **vim で保存して終了した直後の `cat` 出力**
   - `Esc` を押して `:wq` で保存・終了します。
   - 直後に `cat sample.txt` を実行し、保存された内容が表示されている状態を撮影します。

## ヒント

- スクリーンショットにはブラウザのアドレスバーや、モーダル越しにチャット UI が写るようにすると、アプリケーション内で Linux が動作していることが伝わりやすくなります。
- 画像ファイルにはタイムスタンプ付きのファイル名（例: `sandbox-20240701-1.png`）を付けておくと管理が容易です。
- GIF アニメーションとして録画したい場合は `peek` や `OBS Studio` などの録画ツールを利用してください。

## トラブルシューティング

- **noVNC に接続できない**: バックエンドのログに `Sandbox runtime error` が出ていないか確認し、Docker コンテナが起動しているか `docker ps` でチェックしてください。
- **画面が真っ黒なまま**: ネイティブランタイムの場合は `x11vnc` が起動しているか確認し、`SANDBOX_NATIVE_DISPLAY` と `SANDBOX_NATIVE_WIDTH/HEIGHT` の値が適切か見直してください。
- **vim の日本語入力ができない**: 必要に応じて `ibus` や `fcitx` などの IME をコンテナ内に追加インストールしてください。

このチェックリストに従って取得した 5 枚のスクリーンショットを成果物として添付することで、チャットアプリケーション上で Linux 仮想環境が動作していることを明確に示せます。
